Stock Predictor & Paper Trader

A desktop app for LSTM-based Nifty 50 price forecasting and autonomous paper trading — fully offline, no broker required.

Nifty 50 LSTM Forecasting Multi-signal scoring Paper trading only No API key needed

Overview

This app watches the Nifty 50 stock universe, trains a personal LSTM neural-network model per stock, blends that forecast with six independent signals, and either recommends or autonomously executes paper trades — all inside a desktop window. No real money changes hands unless you wire up a real broker later.

🤖 Auto-monitor

Positions are evaluated with fresh LSTM forecasts on a timer while the app is open. SELL recommendations surface automatically.

📊 LSTM Forecasting

A 2-layer LSTM trained on 5-year daily data predicts the price 5 days ahead (swing) or 4 hours ahead (intraday).

⚖️ 6-signal blend

Price forecast, sentiment, sector momentum, fundamentals, macro regime, and diversification — all weighted into one score.

🛡️ Risk guardrails

Stop-loss at −3 %, 75 % portfolio cap, 10 % per-stock cap, and profit goals (min & max) keep risk automatically in check.

Installation & first run

  1. Make sure Python 3.10 + is installed Check with python --version in a terminal.
  2. Install dependencies Open a terminal inside the project folder and run: 
    pip install -r requirements.txt
    This installs PyTorch, yfinance, customtkinter, matplotlib, VADER sentiment, and everything else the app needs.
  3. Launch the app 
    python main.py
    A dark-mode desktop window opens. Two SQLite databases are created automatically on the first run: portfolio.db (your wallet) and accuracy.db (prediction history).
ℹ️

No internet key required. All market data comes from Yahoo Finance (free, no API key). Data is delayed ~15 minutes for NSE — fine for paper trading, not for real intraday execution.

First-time setup

Do these steps once before the app can trade autonomously.

  1. Set your starting capital Go to the Settings tab → Total capital available to invest → enter your paper budget (e.g. ₹1,00,000) → click Set total capital. This refunds the wallet and resets all history.
  2. Choose a trading mode Go to the Agent tab and pick swing (weekly trades) or intraday (same-day trades) from the Horizon selector. Swing is recommended for beginners — it is slower-paced and uses daily data.
  3. Train the LSTM models — click "Train all (first run)" The agent needs a trained model for every stock before it can score or trade. Training downloads 5 years of daily data and fits a neural network per stock.
    ⏱️

    Training all 50 Nifty stocks takes 30 – 60 minutes on CPU (a few minutes on a GPU). You only do this once. Models are saved to model/trained/ and reused on every future run.

  4. You are ready The auto-monitor starts 15 seconds after launch. The agent will evaluate your open positions on every tick. Run a buy scan whenever you want to open new positions.

The 6 signals explained

When the agent scores a stock, it blends six independent signals into one number between −1 (strong sell) and +1 (strong buy). Only stocks with a combined score above 0.15 are considered for buying.

🧠 LSTM forecast
45 %
📰 Sentiment
15 %
🏦 Sector momentum
15 %
📈 Fundamentals
10 %
🏛️ Macro regime
10 %
🔗 Correlation
5 %

🧠 LSTM forecast (45 %)

The model's predicted return for this stock over the chosen horizon. Normalised so a +5 % forecast = score 1.0.

📰 Sentiment (15 %)

VADER sentiment analysis on recent Yahoo Finance news headlines. Positive news → positive score, negative headlines → negative score.

🏦 Sector momentum (15 %)

The 20-day return of the stock's sector index (e.g. ^NSEBANK for bank stocks). Rising sector = positive; falling sector = negative.

📈 Fundamentals (10 %)

Rule-based sanity check on P/E ratio, profit margins, revenue growth, and debt-to-equity. Healthy numbers push the score positive.

🏛️ Macro regime (10 %)

Manually entered in config.py: RBI repo rate direction, inflation vs. target, GDP growth trend. Rate cuts are bullish; high inflation is bearish.

🔗 Correlation (5 %)

A diversification bonus. Stocks that move differently from your existing holdings score higher. Stocks you already hold score −1.

HOLD / SELL decision logic

For each open position the engine asks "should we exit now?" Rules are tested in order — the first match determines the action.

1. Stop-loss hit → SELL Live price ≤ buy price × (1 − stop-loss %). Default: −3 %. Hard limit — always fires.
2. Below breakeven (P&L < 0 %) → HOLD Loss position is held regardless of model outlook. Waiting for recovery to MIN target. Only stop-loss can exit here.
3. Max profit reached → SELL P&L % ≥ MAX target (e.g. +6 % swing). Greed cap — don't push luck.
4. In profit zone [MIN, MAX) → fresh model decides P&L is between MIN and MAX. A fresh LSTM is run:
  • Forecast positive → HOLD (let it run toward MAX)
  • Forecast flat/negative → SELL (lock the gain)
5. Below MIN profit (but positive) → usually HOLD Waiting for the price to reach the MIN target. Exits early only if the fresh model turns strongly bearish (below hold threshold) or the horizon has long expired with no progress.

Position sizing rules

Every potential buy goes through the sizer before the order is placed. Rules are applied in sequence — all must pass.

  1. 75 % portfolio cap invested_cap = equity × 0.75. If adding this position would push total invested above the cap, the trade is skipped.
  2. Edge-driven allocation target % = edge % × allocation_scale (default 4 ×). A 2 % LSTM edge → target 8 % of equity for this stock.
  3. 10 % per-stock cap Capped at 10 % of equity regardless of how large the edge is.
  4. 2 % minimum If the computed allocation falls below 2 % of equity the trade is skipped (too small to be meaningful).
  5. Cash check The rupee cost must fit inside both the available budget (75 % cap headroom) and the current cash balance. Otherwise skipped.

Period goals

The app tracks two profit goals to prevent over-trading once you're ahead.

GoalHorizonPeriodCircuit-breaker effect
Daily goal Intraday Resets every calendar day When reached, the agent stops opening new intraday positions until tomorrow.
Weekly goal Swing Resets every Monday When reached, no new swing buys until the next week starts.

Goal amount = TOTAL_INVESTMENT_AMOUNT × GOAL_TARGET_PCT_OF_CAPITAL (configured in config.py). Progress = realized P&L this period + unrealized P&L on open positions of that horizon.

Workflow — Fully autonomous mode

Set it up once, leave it running, and check back throughout the day.

  1. Complete first-time setup Train all models, set capital, pick horizon (see First-time setup).
  2. Launch the app and keep the window open python main.py — minimising is fine, just don't close it. The auto-monitor runs on its timer as long as the process is alive.
  3. Run a buy scan to open positions Click Run buy scan on the Agent tab. The agent scores all 50 stocks, filters by combined score and correlation, sizes each pick, and buys the top qualifying ones.
  4. Let the monitor work Every N minutes the monitor evaluates each position with a fresh model. When it recommends SELL it logs the symbols in the Agent tab and highlights them in the Portfolio tab.
  5. Execute SELL recommendations Switch to Portfolio & Monitor and click Auto-sell all flagged. Proceeds return to cash.
  6. Repeat step 3 after sells Run another buy scan to redeploy the freed cash into the next best-ranked stocks.
💡

Set the monitor interval to 5 minutes during active trading hours and 60 minutes overnight. The every [N] min field on the Agent tab changes it live.

Workflow — Manual prediction only

Use the app purely as a research tool without autonomous trading.

  1. Open the Predict tab Select a stock and horizon, click Run prediction.
  2. Read the output Check expected return, confidence, and the forecast path on the chart. Compare the MIN/MAX targets to decide if the risk/reward makes sense.
  3. Repeat for other stocks Compare multiple stocks to find the best edge before deciding where to commit capital in your actual brokerage account.

Workflow — Resetting & starting over

Reset the wallet only (keep trade history)

Go to Settings → Reset paper wallet. This closes all positions, refunds the wallet to the default capital, but keeps the full trade log in the Ledger.

Clear trade history (keep open positions)

Go to Ledger → Clear Ledger. Deletes all BUY/SELL records. Open positions and the cash balance are untouched — useful when you want a clean stats baseline without disturbing current holdings.

Full fresh start with new capital

Go to Settings → Total capital → Set total capital. This wipes everything (positions + ledger) and restarts the wallet with the amount you enter.

Delete and retrain a model

Go to Predict tab → pick the stock → click Retrain model. This downloads fresh data and trains a brand-new model, overwriting the old one. Or use Retrain stale models on the Agent tab to retrain every model whose accuracy has slipped.

FAQ

The first prediction takes a long time — is that normal?

Yes. The very first prediction for a stock+horizon trains a new LSTM model from scratch (~1–2 minutes on CPU). After that the model is saved and predictions are near-instant.

Why is the "Run buy scan" not buying anything?

Several reasons are possible:

  • No stocks cleared the combined score threshold (0.15). Market conditions may not favour any Nifty 50 stock right now.
  • The 75 % invested cap is already hit — free up cash by selling some positions first.
  • Models are not trained yet — click Train all first.
  • The period goal has been reached — no new buys until the period resets.

A position is negative — when will it sell?

A negative position is held automatically until it recovers to at least the MIN profit target. It will only sell before then if the stop-loss (−3 % by default) is triggered. You can adjust the stop-loss in Settings.

How do I update macro data (RBI rate, inflation)?

Open config.py in a text editor, find the MACRO_INPUTS dictionary, and update the values. Restart the app for changes to take effect. Do this after every RBI MPC announcement or major economic data release.

Can I use this for real money?

The app is paper-only by design. The real broker interface (trading/broker_interface.py) is a stub that raises NotImplementedError. Before live trading, run at least 100 paper trades over 4+ weeks and verify: win rate > 40 %, max drawdown < 10 %, profit factor > 1.5. Only then implement the broker adapter with your Zerodha / Upstox credentials.

The model says SELL but I disagree — can I override?

Yes. The auto-monitor only recommends — it never executes without you clicking Auto-sell all flagged. You can simply ignore the recommendation and the position stays open. Use Refresh decisions later to get an updated verdict.

The app is slow / freezing

Heavy operations (LSTM training, scoring 50 stocks) run in background threads so the UI stays responsive. If the window appears stuck, check the Agent tab log — a progress message should appear. On CPU-only machines training all 50 stocks can take 60 minutes; this is expected.
